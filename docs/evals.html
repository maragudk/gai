Evaluations

GAI provides tools to evaluate model outputs. Test how well models perform on your specific tasks.

Why Evaluate Models

Evaluations help you:

  • Compare model performance
  • Track quality over time
  • Identify areas for improvement
  • Verify behavior on edge cases
  • Make informed provider choices

Model evaluation is often overlooked but critical for AI applications. A systematic evaluation approach helps you quantify model quality, track regressions, and make data-driven decisions about which models to use.

Core Evaluation Concepts

GAI's evaluation system uses:

Samples

Input, output, and expected output triplets

Scorers

Functions that evaluate outputs against expectations

Results

Score and metadata from evaluation

Logs

Structured records of evaluations

Each sample represents a test case with an input query, the model's actual output, and the expected (reference) output. Scorers calculate how well the actual output matches the expectation, producing a normalized score between 0 and 1.

Running Evaluations

GAI runs evaluations as part of normal Go tests:

package myapp_test

import (
    "context"
    "testing"

    "maragu.dev/gai"
    "maragu.dev/gai/eval"
)

// TestEvalAccuracy evaluates model output accuracy.
// Must be prefixed with "TestEval" to be detected.
func TestEvalAccuracy(t *testing.T) {
    eval.Run(t, "capital cities accuracy", func(t *testing.T, e *eval.E) {
        // Initialize model client
        model := initializeModel()

        // Send request to model
        input := "What is the capital of France?"
        res, err := model.ChatComplete(t.Context(), gai.ChatCompleteRequest{
            Messages: []gai.Message{
                gai.NewUserTextMessage(input),
            },
        })
        if err != nil {
            t.Fatal(err)
        }

        // Collect response
        var output string
        for part, err := range res.Parts() {
            if err != nil {
                t.Fatal(err)
            }
            output += part.Text()
        }

        // Create evaluation sample
        sample := eval.Sample{
            Input:    input,
            Output:   output,
            Expected: "Paris is the capital of France.",
        }

        // Score using lexical similarity
        lexicalResult := e.Score(sample, eval.LexicalSimilarityScorer(eval.LevenshteinDistance))

        // Score using semantic similarity
        semanticResult := e.Score(sample, eval.SemanticSimilarityScorer(t, model, eval.CosineSimilarity))

        // Log results
        e.Log(sample, lexicalResult, semanticResult)
    })
}

Key Steps

  1. Create a test function with a TestEval prefix
  2. Use eval.Run to execute the evaluation
  3. Send a request to your model and collect its response
  4. Create a Sample with input, output, and expected results
  5. Apply one or more scoring methods
  6. Log the results for analysis

Running Eval Tests

Evaluations only run when you explicitly include them:

# Run all evaluations
go test -run TestEval ./...

# Run specific evaluations
go test -run TestEvalAccuracy ./...

Note

Evaluations are skipped during normal test runs to avoid unexpected API calls. You must explicitly run them with the TestEval pattern in the test name filter.

Scoring Methods

GAI includes multiple scoring methods:

Lexical Similarity

Compare text strings directly:

// Using Levenshtein distance
lexicalResult := e.Score(sample, eval.LexicalSimilarityScorer(eval.LevenshteinDistance))

// Exact match (1.0 if identical, 0.0 otherwise)
exactResult := e.Score(sample, eval.LexicalSimilarityScorer(eval.ExactMatch))

// Contains match (1.0 if output contains expected, 0.0 otherwise)
containsResult := e.Score(sample, eval.LexicalSimilarityScorer(eval.Contains))

Levenshtein Distance

Measures character-level edits needed to transform one string into another

Exact Match

Binary scoring - 1 if strings match exactly, 0 otherwise

Contains

Binary scoring - 1 if output contains the expected string, 0 otherwise

Semantic Similarity

Compare meaning using embeddings:

// Using cosine similarity on embeddings
semanticResult := e.Score(sample, eval.SemanticSimilarityScorer(t, model, eval.CosineSimilarity))

Semantic Similarity

This technique converts both output and expected text to embeddings, then measures their similarity in vector space. It's more robust than lexical scoring for recognizing paraphrased content with the same meaning.

LLM Judge

Use another LLM to judge outputs:

// Basic judge using default prompt and temperature
judge := eval.DefaultJudge(gai.Temperature(0.1))
llmResult := e.Score(sample, eval.LLMScorer(t, judge, judgeModel))

// Rubric-based judge for multi-dimensional evaluation
rubricJudge := eval.RubricJudge(gai.Temperature(0.1))
rubricResult := e.Score(sample, eval.LLMScorer(t, rubricJudge, judgeModel))

LLM Judges

LLM judges can evaluate outputs in a more nuanced, human-like way. The RubricJudge even provides multi-dimensional scoring across content accuracy, relevance, completeness, and clarity.

Evaluation Groups

Group evaluations for comparison:

e.Group = "FactualAccuracy"

Using Groups

Groups help you organize evaluations by category, making it easier to analyze results. Common groupings include:

  • Task type (summarization, QA, etc.)
  • Evaluation criteria (accuracy, creativity, etc.)
  • Model being tested
  • Data source or domain

Timing Information

Reset timer to focus on specific operations:

// Reset timer before evaluated operation
e.ResetTimer()

// Perform operation to time
output := model.GenerateResponse(input)

// Log with timing information
e.Log(sample, result)

Timing Tip

Use ResetTimer to exclude setup time from your measurements. This gives you more accurate timing of just the operation you're interested in, similar to benchmarking in Go.

Real-World Example

Comprehensive evaluation suite:

func TestEvalSummarization(t *testing.T) {
    eval.Run(t, "article summarization", func(t *testing.T, e *eval.E) {
        // Load test articles
        articles := loadTestArticles()
        
        // Test each article
        for _, article := range articles {
            // Reset timer for fair performance comparison
            e.ResetTimer()
            
            // Get model summary
            res, err := model.ChatComplete(t.Context(), gai.ChatCompleteRequest{
                Messages: []gai.Message{
                    gai.NewUserTextMessage("Summarize this article: " + article.Content),
                },
            })
            if err != nil {
                t.Fatal(err)
            }
            
            // Collect response
            var summary string
            for part, err := range res.Parts() {
                if err != nil {
                    t.Fatal(err)
                }
                summary += part.Text()
            }
            
            // Create sample with article's human-written summary as expected
            sample := eval.Sample{
                Input:    article.Content,
                Output:   summary,
                Expected: article.HumanSummary,
            }
            
            // Score with multiple methods
            semantic := e.Score(sample, eval.SemanticSimilarityScorer(t, model, eval.CosineSimilarity))
            llm := e.Score(sample, eval.LLMScorer(t, eval.RubricJudge(0.1), judgeModel))
            
            // Log results
            e.Log(sample, semantic, llm)
        }
    })
}

Best Practices

  • Use multiple scoring methods to get a more complete picture
  • Test against a diverse set of examples
  • Reset the timer for fair comparisons between operations
  • Log detailed information for later analysis
  • Consider both automated metrics and human-like (LLM) judgments

Visualization

GAI writes evaluation results to evals.jsonl in your project root.

You can visualize results with:

  • Custom dashboards
  • JSONL processors
  • Visualization libraries

Web Dashboard Integration

// Configure data export to dashboard service
os.Setenv("EVALS_API_KEY", "your-api-key")
os.Setenv("EVALS_PROJECT", "your-project")

// Run evals as normal - will automatically upload results
go test -run TestEval ./...

Results appear at https://api.evals.fun/evals.svg?key=your-api-key
